KMSA – Karlchen Modular System Architecture

Architektur für Production-Ready Modular Systems. Prinzip: Es werden Module (Packages) angelegt, die ausschließlich über SharedCore kommunizieren – keine direkte Kopplung zwischen Modulen.

Kernfeatures

  • Module (Packages) anlegen: Jedes Modul wird als eigenes Package definiert und meldet sich bei SharedCore an
  • Kommunikation nur über SharedCore: Module kennen sich nicht untereinander, nur über SharedCore; 1 Modul kaputt, Rest läuft
  • Auto-Discovery: Modul erstellen → Bridges anlegen → SharedCore registriert automatisch
  • Self-Healing: ModuleHashHistory, Rollback bei kompromittiertem Hash (z. B. Ransomware)
  • Observability & Forensik: ModuleConfiguration, ModuleStatusHistory, KMSADebugLogger – Audit-Trail
  • Security: AIModelSigningService (Curve25519), AdvancedEncryption AES-GCM 256-Bit, EncryptedKeyStorage
