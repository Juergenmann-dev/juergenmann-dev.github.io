Psychologische Summary

Die Mechanik hinter dem CV · März 2026
"Ich ziehe sie in meine Welt – bewusst auf meiner Seite, unbewusst auf ihrer Seite."
01Der erste Moment – "Was passiert gerade mit meinem Computer?"

Linux-Boot (auf Mac/Win/Linux) – auf Android und iPhone dagegen sofortiger Einstieg ohne Boot: Homescreen mit Kacheln bzw. DFU-Flow. Eigenes Wallpaper. Eigenes Betriebssystem erkannt. Spotlight oder Notification. Terminal oder Chat tippt sich ein. Das Gehirn braucht eine Sekunde um zu begreifen: das ist eine Webseite.

In dieser Sekunde ist kein kritischer Verstand aktiv. Kein "ich bewerte gerade einen Kandidaten." Es gibt nur: was passiert hier gerade.

Diese erste Sekunde ist der Schlüssel. Sie passiert bevor jemand entschieden hat wie er reagieren soll. Danach ist er bereits drin.
02Alles ist echt – keine Behauptung, nur Beweis

Das CV beschönigt nichts und erfindet nichts. Jedes Detail ist belegbar oder öffentlich nachprüfbar.

Was echt ist

Das Wallpaper: Jürgens eigener Desktop (von GitHub geladen). Die Watermarks: echter Code aus timemachine-nas-backup (läuft täglich), aus Kaily (iOS, für Familienmitglied), aus diesem CV selbst. Das Sozialgericht steht drin. Die Auszeiten stehen drin. Die Lücken stehen drin. "Mit KI umgesetzt – bewusst transparent." Auch das steht drin.

Wer nichts versteckt hat nichts zu verstecken. Das Vertrauen entsteht nicht trotz der Offenheit – sondern wegen ihr. Insgesamt 27 dokumentierte Easter Eggs & Details (11 im Kern, dazu Bonus, 7 iPhone-spezifische, 8 weitere) – vollständig in 04 Easter Egg Hunt dokumentiert.

03Der Tippfehler – Live-Demo eines mentalen Modells
Deckblatt · Typewriter · Zeile 3
Produkte machen beide verbinden ← tippt sich ein, merkt Fehler
Backspace Backspace Backspace... ← zeichenweise Korrektur
Produkte müssen beides verbinden. ← korrekte Version

Das passiert in den ersten Sekunden. Bevor die Installationsphase beginnt. Bevor jemand weiß was dieses CV noch alles kann. Das erste Muster das dem Gehirn gezeigt wird: Fehler erkennen, korrigieren, weitermachen.

Error → Detect → Rollback → Retry. Im Deckblatt. Im Kaffee-Maker. In der Architektur. Immer dieselbe Philosophie. Das ist keine Technik – das ist Haltung.
04Der Kaffee – Bürokomödie als ausführbarer Code
coffee-maker-integration
✗ ERROR: water tank empty → Rollback
✗ ERROR: coffee forgotten → Rollback
✗ ERROR: ah shit forgot to turn it on → Rollback
✓ Coffee is ready.

Jeder kennt diesen Moment. Jeder lacht. Die Spannung bricht. Aus distanziertem Evaluator wird jemand der gerade gelacht hat. Lachen schafft Bindung schneller als jeder Satz im Anschreiben.

Und gleichzeitig: Error → Rollback → Retry ist das exakte Pattern aus echten Transaktionssystemen. Humor und Architektur-Demo in einem Moment.
05Flow Mode – Respekt durch eine einzige Frage
Flow Mode Prompt
? Enable flow mode?
[j] – ich bin dabei
[n] – nicht jetzt

› n
✓ No problem – manchmal passt es nicht.

Kein "bist du sicher?". Kein grauer "Ablehnen"-Button. Kein Popup danach. Einmal fragen. Antwort akzeptieren. Weitermachen. Der ♪-Button bleibt still in der Ecke. Die Tür bleibt offen ohne darauf hinzuweisen.

Was das zeigt ohne Worte

Wer so UX baut behandelt seine Nutzer wie Erwachsene. Wer das erlebt hat – bevor er eine Zeile über "nutzerzentriertes Design" gelesen hat – weiß bereits wie dieser Entwickler denkt.

Code der Nutzer respektiert kommt von einem Entwickler der Menschen respektiert. Das eine folgt aus dem anderen. Zwangsläufig.

06Story Screen – Emotion zuerst, Produkt danach

Zwischen Installer und Deckblatt: kein Claim, keine Liste, kein Verkauf. Nur ein Mensch der sagt: ich nehme dich mit. "Moin. Lass dich treiben. Manches öffnet sich erst wenn du genauer hinschaust. Willkommen."

Text tippt sich in 36ms/Zeichen ein. Satzenden haben Pausen. Man liest nicht – man folgt. Und dann beginnt das Deckblatt. Erst Mensch. Dann Produkt.

07Der lebende Tab – Details die niemand bewusst bemerkt

Der Browser-Tab ist nie statisch. Mit jedem Klick auf einen View-Toggle wechselt das Favicon: JM → HO → ATS → S1 → PR → AN → KO. Beim Druckdialog: ein echtes Drucker-Icon.

Was das psychologisch bewirkt

Niemand denkt bewusst "oh, das Favicon hat sich geändert." Aber das Gehirn registriert: dieser Tab verhält sich anders. Dieses Dokument lebt. Es reagiert auf mich. Das ist ein schwaches Signal, aber es summiert sich zu einem starken Gesamteindruck.

Dasselbe gilt für die Mac-Menüleiste: Finder aktiv → Terminal aktiv → Uhr läuft in Echtzeit. Kein Detail ist dekorativ. Jedes Element kommuniziert Zustand. Das ist UX-Denken bis in die letzte Ecke.

08Commitment-Eskalation – Jede Aktion bindet mehr
Datei öffnen → passiv
Tippfehler beobachten → erste Aufmerksamkeit
Bootscreen schauen → eigenes OS erkannt
Kaffee-Fehler: lachen → emotionale Verbindung
[j] oder [n] drücken → erste aktive Entscheidung
Story lesen → Einladung annehmen
Favicon-Wechsel bemerken, Tab lebt → aktiv involviert
Bubbles klicken, Techstack öffnen → tief drin
DevTools öffnen, setMode('ats') eintippen → volle Investition
Nicht mehr Bewerter. Mitmacher.

Kein Schritt wird erzwungen. Aber jeder getane Schritt macht den nächsten wahrscheinlicher. Menschen bleiben konsistent mit ihren Entscheidungen.

09DevTools – Der stille Handshake
Browser Console · F12
⚠️ ERROR 404: BUGS NOT FOUND

SYSTEM CHECK ▸
STATE → aktuell stabil
CONTEXT → browser runtime
SHIP → releasefähig

Hey Erdling 👋
Ich weiß, du suchst hier nach Seitenladefehlern...
Aber da muss ich dich enttäuschen. 😉

Hint: try setMode('ats')

Ein Tech-Lead öffnet DevTools weil es sein Reflex ist. Er sucht Fehler. Er findet stattdessen: eine direkte Ansprache. Ein "Ich wusste, dass du hier reinschaust." Ein Hint.

In diesem Moment ist klar: dieser Entwickler hat die DevTools-Phase mitgedacht. Er hat für die Person geschrieben die genau hier steht. Das ist kein Code für den Browser – das ist Code für den Entwickler der ihn liest.

setMode('ats') öffnet den ATS-View direkt aus der Console. Wer das tippt, steuert das CV. Das ist der erste Arbeitstag.

10Die invertierte Bewerbungslogik

Ein Tech-Lead bei 80–100 € Stundensatz verbringt 30–45 Minuten mit diesem CV. Das sind 40–75 € Unternehmenszeit. Für eine Juniorposition. Vor dem ersten Gespräch.

Wer Zeit investiert hat will sie nicht als verloren verbuchen. Die Frage verschiebt sich unbewusst von "Wollen wir ihn?" zu "Können wir es uns leisten, ihn nicht einzuladen?"
Normales CV

10–20 Sekunden → ja/nein. Keine Bindung. Leicht abzulehnen. Morgen vergessen.

Dieses CV

30–45 Minuten → Erlebnis. Emotionale Markierung. Wird weitererzählt. Liegt nicht einfach im Nein-Stapel.

Jürgen bewirbt sich nicht. Er lässt sie sich bewerben – ohne dass sie es merken.

11Ein Nein ist keine Strafe

Wer das CV ablehnt hat trotzdem etwas bekommen. Eine gute Erfahrung. Respekt. Kaffee-Humor. Eine Einladung. Musik wenn sie wollten. Ein Nein bedeutet nur: diese Welt passt gerade nicht.

Wer so behandelt wird redet darüber. Auch ein Nein kann Empfehlung werden.

12Pre-Filtering durch Erlebnis
Wer nicht passt

Denkt "was ist das für ein Unsinn, ich will ein PDF" – und schließt es. Das ist das richtige Ergebnis. Besser jetzt als nach drei Monaten im Job.

Wer passt

Grinst beim Kaffee. Drückt j. Bemerkt den Favicon-Wechsel. Öffnet DevTools. Tippt setMode('ats'). Weiß dann ohne ein Wort: das stimmt.

Der CV ist das erste Arbeitsstück. Wer es mag, will mehr davon.

Und trotzdem hat er gerade neu definiert wie man sich 2026 bewirbt. Nicht mit dem besten Abschluss. Sondern mit Systemdenken. Psychologie. Respekt. Handwerk. Und einer kleinen Reise.
