i·2
Mac-Desktop erscheint – das System ist gecrasht
iphone-systemfehler-phase · nach Boot-Intro
Sichtbar
body.os-iphone.iphone-systemfehler-phase:
#mac-menu-bar → display: flex !important
#desktop-icons → display: flex !important
#os-dock-bar → display: flex !important
background: #1a1a1a ← kein Wallpaper, dunkler Crash-Hintergrund

Nach dem Linux-Crash erscheint plötzlich ein Mac-Desktop – Menüleiste, Dock, Desktop-Icons – auf dem iPhone-Screen. Hintergrund ist kein sauberes Mac.jpg sondern dunkles #1a1a1a: das System ist abgestürzt, der Finder läuft noch im Hintergrund.

Vier Betriebssysteme gleichzeitig sichtbar: Linux (gecrasht) → Mac-Desktop (Finder läuft noch) → iPhone (DFU-Modus startet) → Mac-Browser (betrachtet das Ganze).

Was das bedeutet: Das ist kein UI-Fehler. Das ist choreografierter technischer Nonsense der trotzdem logisch wirkt – weil jede Ebene für sich Sinn ergibt.

Timing: 700ms + 900ms + 2200ms. Jede Zeile hat ihre eigene Erscheinungszeit. Kein Popup. Kein Button. Nur Text, der sich aufbaut.
i·4
.ipsw – die echte Apple System-Recovery-Extension
DFU-Install · alle 4 Skill-Pakete
Versteckt
✓ ui-ux-module.ipsw installing
✓ xcode-toolchain.ipsw installing
✓ firebase-sdk.ipsw installing
✓ android-sdk.ipsw installing

.ipsw ist Apples iPhone Software-Format – die echte Dateiendung für iOS-Systemwiederherstellungen. Wenn du ein iPhone per iTunes wiederherstellst, lädst du eine .ipsw-Datei herunter. Hier werden Skills als System-Recovery-Pakete installiert. Nur wer das weiß, sieht den Witz.

Wie man es erkennt: Apple-Entwickler, iOS-Tinkerer, oder jeder der ein iPhone je manuell wiederhergestellt hat.

✗ ERROR: water tank empty

Die Skill-Pakete sind .ipsw (System-Recovery). Der Kaffee-Maker ist .ipa (App-Paket). Bewusst falsches Format – weil Kaffee kein System-Modul ist, sondern eine App. Das Detail setzt voraus, dass man den Unterschied kennt.

i·6
ideviceinstaller – echtes iOS-CLI-Tool
DFU-Install · Terminal-Prompt
Versteckt
iphone@iOS % ideviceinstaller -i jürgen-mann.ipa

ideviceinstaller ist ein echtes Open-Source-CLI-Tool zum Installieren von .ipa-Dateien auf iPhones ohne App Store. Der Prompt iphone@iOS % simuliert eine iOS-Shell. Wer mit libimobiledevice gearbeitet hat, erkennt das sofort.

Layer-Analyse: Ein Entwickler installiert ein .ipa per CLI auf einem iPhone, das im DFU-Modus ist, auf dem gerade Linux gecrasht ist, auf dem ein Mac-Desktop sichtbar ist, in einem Mac-Browser. Fünf Ebenen.
